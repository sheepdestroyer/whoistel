The history database file path is hardcoded. This makes the application less flexible for different deployment environments or for running certain kinds of tests. It's better to make this configurable via an environment variable, with the current path as a default. This allows overriding the database path without changing the code. Remember to add `import os` at the top of the file.

```suggestion
DB_FILE = os.environ.get('HISTORY_DB_FILE', 'history.sqlite3')
```
